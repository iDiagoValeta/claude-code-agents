▌ MEMORIA (CLAUDE.md)

Jerarquía de ubicaciones

./CLAUDE.mdInstrucciones del proyecto actual
./.claude/CLAUDE.mdAlternativa del proyecto
~/.claude/CLAUDE.mdPreferencias personales globales
/etc/claude-code/CLAUDE.mdPolítica de organización (admin)

Importar archivos en CLAUDE.md

@READMEImporta el README del proyecto
@package.jsonImporta package.json
@docs/guia.mdImporta archivo específico
@~/.claude/prefs.mdImporta preferencias personales

Reglas por ruta (frontmatter YAML)

---
paths:
  - "src/api/**/*.ts"
  - "src/**/*.{ts,tsx}"
---
# Reglas bajo este frontmatter solo aplican a esos paths

Buenas prácticas

  • Máx. ~200 líneas por archivo (mejor adherencia del modelo)
  • Usar headers y bullets para estructurar
  • Instrucciones específicas y verificables
  • Evitar contradicciones entre archivos
  • Usa /init con CLAUDE_CODE_NEW_INIT=true para flujo interactivo completo
▌ FLAGS CLI

Modelo y esfuerzo

--model sonnet|opusAlias o nombre completo del modelo
--effort low|medium|high|maxNivel de esfuerzo (max = Opus 4.6)
--fallback-model sonnetFallback si modelo sobrecargado (-p)
--betas interleaved-thinkingBetas de API (solo API key)

Control de ejecución

--permission-mode planSolo planificar, sin ejecutar
--max-turns NLimitar turnos del agente (modo -p)
--max-budget-usd 5.00Límite de gasto en USD (modo -p)
--dangerously-skip-permissionsOmitir confirmaciones (⚠ cuidado)
--tools "Bash,Edit,Read"Restringir herramientas disponibles
--disallowedTools "Edit"Eliminar herramientas del contexto
--allowedTools "Bash(git*)"Auto-aprobar herramientas específicas
--disable-slash-commandsDeshabilitar todos los skills
--bareModo mínimo sin hooks/MCP/memoria
--no-session-persistenceSesión efímera sin guardar (-p)

System Prompt

--system-prompt "..."Reemplazar system prompt completo
--system-prompt-file file.txtReemplazar prompt desde archivo
--append-system-prompt "..."Añadir al system prompt (recomendado)
--append-system-prompt-file fAñadir desde archivo al prompt
--system-prompt y --system-prompt-file son mutuamente exclusivos

Sesiones y salida

--input-format text|stream-jsonFormato de entrada (con -p)
--json-schema '{...}'JSON Schema para output validado
--include-partial-messagesIncluir eventos de streaming parciales
--verboseLogs detallados turno a turno
--debug "api,mcp"Debug con filtros de categoría
--worktree / -w nombreIniciar en git worktree aislado
--fork-sessionFork al reanudar (nuevo ID)
--name / -n "nombre"Asignar nombre a la sesión
--session-id "uuid"Usar ID de sesión específico

▌ FLUJOS DE TRABAJO

Inicio de proyecto

claude --initInit interactivo (CLAUDE.md, hooks, skills)
claude -w feature-authSesión paralela en worktree aislado
claude --permission-mode planSolo planificar, sin ejecutar código
claude -n "auth-refactor"Sesión nombrada para retomar luego

Pipelines no interactivos (modo -p)

claude -p --max-turns 5 --output-format json "query"
cat error.log | claude -p "explica y corrige"
claude -p --json-schema '{...}' "genera objeto validado"
claude -p --max-budget-usd 2 --fallback-model sonnet "q"
claude --bare -p "query"  # arranque mínimo y rápido
